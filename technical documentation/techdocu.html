OOps definition

Oops stands for object oriented programming language, This is in contrast to the tradition procedural programming language where we look at the program as a set of instruction and try to achieve that with the help of function where the functions are used to get the tasks done.

Here we look at the problem as a set of objects which have the datamembers and the functions which act upon them.

OOPS Advantage

The major advantage of OOps over traditional procedural programming is that it provides us with a number of features which make the software development task easy to model and understand and thus reduces the complextiy and cumbursumeness. It provides a number of special features which is not available in the procedural programming, that includes but is not limited to data hiding feature which puts a restraint on the number of people who can acces and change it, while helping the ones who have the required authentication to do that.

The othe one is the concept of class as they provides encapsulation which means that the data and the functions reside together and the are thus closely related to one anothe as opposed to the prodedural programming There are infact many advantages and benefits which we will take a look as we go deep into this article.

Inheritance

Inheritance is the property of oops which truly promotes the concept of reusability, some times the kind of data and member functions that we need are already present in the class and we just need to modify it a little in that kind of cases , inheritance can be a boon for us, the way of using inheritance is as follows:-

class B: public A

where public defines the visibility mode

Types of inheritances are:-

  • single level inheritance
  • multi level inheritance
  • heirarchial inheritance
  • hybrid inheritance
